## Technical Details

The vulnerability is documented in the source code with the following key information:

<Info>
  **CVE ID:** CVE-2023-27372  
  **Vulnerability Type:** Unauthenticated Remote Code Execution via PHP Object Injection  
  **Attack Vector:** The `oubli` parameter in `/spip.php?page=spip_pass` (password reset form)  
  **Root Cause:** Mishandled deserialization of user-supplied serialized data
</Info>

### Exploitation Mechanism

From the source code analysis:

1. The vulnerability exploits improper deserialization in the `oubli` parameter
2. A serialized PHP payload is injected into the password reset form
3. The payload is executed server-side as the web user
4. Command output is reflected back in the response inside the `oubli` input field value

```python
# Example payload structure from source code
payload = f's:{20 + len(command)}:"<?php system(\'{command}\'); ?>";'
```

The exploit extracts output using regex pattern matching:

```python
match = re.search(r'value="s:\d+:"(.*?)";"\s', r.text, re.DOTALL)
```
